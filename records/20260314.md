# CrowdModels 进展记录

更新时间：2026-03-16

## 已完成

1. 模型方法文档已整理
- 已在 methodology 目录形成 `model.md`、`model.html`、`model.pdf`，包含连续介质人群模型、M(x) 各向异性引导和 HJB 控制约束框架。

2. 基线方法（Hughes + 动态势场）已实现并跑通
- 脚本：`codes/simulate_scene1.py`
- 核心能力：场景构建、拥挤代价势场重计算、速度场更新、迎风通量连续性更新。
- 已支持 A/B 两个 case（B 中对中间通道做“禁止向右”限制）。
- 当前产物：`codes/results/scene1` 下 400 个结果文件。

3. 方法2（仅 M 张量几何引导）已实现并跑通
- 脚本：`codes/simulate_method2_scene2.py`
- 核心能力：各向异性 eikonal（对角 M）求解、按 M 修正方向场、连续性方程耦合更新。
- 当前产物：`codes/results/scene1_metricA` 下 200 个结果文件。

4. 方法3（M + HJB 控制集）已实现并跑通
- 脚本：`codes/simulate_method3_scene1.py`
- 核心能力：
	- 控制集 U(x) 离散化（中间通道限制 +x 方向）；
	- 基于值迭代的离散 HJB 势函数；
	- 由 argmin 选取最优方向 u*；
	- 增加 CFL 限制提升数值稳定性。
- 当前产物：`codes/results/scene1_M_plus_HJB` 下 200 个结果文件。

## 当前状态判断

- 结果目录已形成阶段性样本，具备进入对比分析与写作的条件。

## 下一步建议

- 例如：总疏散时间、通道利用率（上/中/下占比）、拥堵峰值密度、出口流率曲线。

2. 规范命名与实验标签
- 当前存在命名不一致风险（如脚本名包含 scene2，但输出目录为 scene1_metricA），建议统一“场景-方法-参数”命名规则。

3. 形成可复现实验批处理
- 固定随机种子/参数表，批量导出关键帧与汇总表，便于写作与复核。

4. 开始写作骨架
- 在 writing 目录下：问题定义、方法差异、实验设置、结果讨论、局限与后续工作.
