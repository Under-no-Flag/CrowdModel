## 2. G4 对比实验与负结果分析

### 2.1 对比实验实现

本周首先补齐了 G4 的正式对比实验，避免只报告单一 SA-HBO 结果。实验使用统一场景、统一目标函数和统一输出根目录，将以下方法纳入比较：

| 方法 | 作用 |
|------|------|
| baseline | 固定全双向开放和默认 `eta=(8,8,8,8)`，作为无优化参考 |
| random search | 在方向配置和 `eta` 空间中随机采样，检验低维空间下的朴素强基线 |
| grid search | 低维粗网格搜索，提供可解释的传统枚举基线 |
| pure SA | 不使用代理筛选的模拟退火式结构搜索 |
| SA-HBO no proxy | 去掉代理模型后的 SA-HBO 消融 |
| SA-HBO | 当前论文主方法的精简实现 |

主要输出目录为：

```text
codes/results/g4_minimal_matrix/
```

该目录中生成了 `g4_visual_summary.md`、`g4_top_candidates.csv`、方法对比图、目标轨迹图、Pareto 图、方向矩阵、eta 热力图和通道流量热力图等产物，可直接支撑论文实验章节或后续复盘。

### 2.2 G4 主要结果

在 `codes/results/g4_minimal_matrix/` 中，共记录 101 次评估。方法对比结果如下：

| 方法 | 最优目标值 | 评估次数 | 最优方向配置 |
|------|------:|------:|------|
| baseline | 0.680543 | 1 | `FREE,FREE,FREE,FREE` |
| random search | 0.560400 | 20 | `W,E,FREE,E` |
| grid search | 0.611446 | 20 | `FREE,FREE,FREE,FREE` |
| pure SA | 0.580038 | 20 | `W,E,E,FREE` |
| SA-HBO no proxy | 0.584554 | 20 | `FREE,E,W,FREE` |
| SA-HBO | 0.585680 | 20 | `FREE,E,W,FREE` |

结果说明：

- SA-HBO 优于 baseline 和 grid search，但没有超过 random search 与 pure SA。
- random search 在当前低维混合空间中非常强，20 次评估即找到目标值 `0.560400` 的最优候选。
- SA-HBO 和 no-proxy 版本都收敛到 `FREE,E,W,FREE` 一类方向配置，说明当前结构邻域和代理筛选没有打开足够大的搜索空间。
- 论文中不能继续直接写成“SA-HBO 显著优于传统优化算法”，更准确的表述应是“当前结构感知优化框架可行，但在该低维场景下未超过随机搜索，仍需改造控制变量和优化机制”。

### 2.3 问题分析记录 `20260514.md`

基于 G4 对比结果，本周撰写了 `records/20260514.md`，集中分析三个问题。

第一，`J2` 明显小于 `J1` 和 `J5`。原 `J2` 是硬阈值高密度暴露：

```text
J2 = int 1_{rho > rho_safe} dxdt
```

它只记录是否超过阈值，不记录超过多少，导致大量中等拥堵状态对安全项贡献为 0。从 G4 结果看，baseline 的 `J2≈0.005112`，random search 最优的 `J2≈0.005686`，而 `J1≈0.45`、`J5≈0.10-0.22`，等权加和时安全项几乎不驱动搜索。因此记录中提出将 `J2` 改为 soft 风险密度：

```text
J2_soft = int [((rho-rho_safe)_+) / rho_safe]^gamma dxdt
```

第二，`eta` 连续变量对目标函数作用弱。G4 优秀解之间的目标差异主要来自方向配置 `s` 和通道负载均衡 `J5`，而不是 `eta`。SA-HBO 的最优 `eta` 基本仍在 `(8,8,8,8)` 附近，说明连续块优化没有真正发挥作用。

第三，当前 SA-HBO 的离散搜索半径和代理筛选不足。random search 的最优方向 `W,E,FREE,E` 与 SA-HBO 的 `FREE,E,W,FREE` 差异较大，说明半径为 1 的局部结构搜索很难跨越多个通道状态变化；代理筛选候选太少时，也容易把真实优良候选挡在全仿真之外。

这一分析直接推动了后续模型从 `eta` 转向入口容量 `q`。

## 3. 文献调研与 Model v2 改造

### 3.1 入口流入速率控制调研

针对 `eta` 作用弱的问题，本周调研并撰写了：

```text
references/连续介质人群模型中通道入口流入速率控制的前沿文献与对当前模型的改造建议.docx
```

该报告的核心判断是：连续介质/宏观人群模型中，真正有效且可实施的管控变量通常更靠近“边界、入口、节点和设施”，而不是继续微调域内连续张量参数。相关文献主要集中在三类思路：

1. 将入口作为混合边界上的给定通量或受控通量；
2. 将通道交汇、入口和阶段切换表示为 sending-receiving 供需约束与 turning fraction；
3. 使用势函数、局部吸引或 Logit 路径代价间接影响人群去向。

该调研给出的直接建议是：

- 不应继续把 `eta` 作为主创新控制变量；
- 保留度量张量 `M(x;eta0)`，但将其降级为几何/各向异性结构参数；
- 将主动控制变量改为入口流入速率 `q_c^\pm(t)`，必要时扩展到阶段分流概率 `p_{g->h}`；
- 入口限流应通过有限体积通量项实现，而不是直接覆写单元密度；
- 被限流的人群不能消失，应通过等待区质量或缓冲队列记录；
- 控制变量应采用低维分段常值形式，降低优化维数并贴近现实闸机、栏杆、分时放行等管理动作。

### 3.2 `model v2.md` 的模型重构

在上述调研基础上，本周重写并完善了：

```text
methodology/model v2.md
```

V2 模型的核心变化是将旧优化变量：

```text
z_old = (s, eta)
```

改为：

```text
z_v2 = (s, q)
```

其中：

- `s` 仍表示通道方向配置，包括正向、反向、双向和关闭；
- `q={q_c^+(t), q_c^-(t)}` 表示每条通道两个方向入口上的通行速率上限；
- `eta` 不再参与优化，只保留固定值 `eta0`，作为通道内部几何引导机制。

文档中补齐了完整的符号表和理论定义，重点区分了三类流量概念：

| 符号 | 含义 |
|------|------|
| `q_c^\pm(t)` | 控制变量，入口允许通行速率上限 |
| `A_c^\pm(t)` | 尝试进入通道的自然流量 |
| `Ahat_c^\pm(t)` | 限流后实际允许通过量 |

V2 模型还明确了方向与速率的一致性约束：

- `E` 方向只允许 `plus` 入口容量，反向容量为 0；
- `W` 方向只允许 `minus` 入口容量，正向容量为 0；
- `FREE` 允许双向容量，但满足双向总容量约束；
- `CLOSED` 使两个方向容量均为 0。

在数值实现上，`model v2.md` 明确入口速率控制是内部通道入口截面的通量约束，而不是计算域外边界 Neumann 入流条件。有限体积更新中应先计算自由通量，再按：

```text
Ahat_c^\pm(t) = min(A_c^\pm(t), q_c^\pm(t))
```

缩放实际通量，从而保持质量守恒。

同时，文档新增了两个对 V2 很关键的目标项：

- `J_B`：入口等待或排队暴露，用于防止优化器通过过度限流把拥堵转移到入口外；
- `J_R`：容量曲线平滑性，用于约束管控动作频繁跳变，提高可实施性。

## 4. Optimization v2 调研与算法方案

### 4.1 算法选择报告

为了给 `z=(s,q)` 设计合理优化算法，本周继续调研并撰写：

```text
references/基于前沿与权威文献的 Bellman—守恒律人群管控优化建模与算法选择报告.docx
```

报告将当前问题定位为两类问题的交叉：

1. 数学结构上接近混合整数 PDE 约束优化；
2. 工程实现上接近昂贵仿真驱动的混合变量黑盒优化。

报告的结论是：当前不应优先把问题作为通用 MINLP 直接交给 branch-and-bound 或 outer approximation 求解器。原因是 Bellman 方程中的 `min/argmin`、入口通量裁剪 `min(A,q)`、安全指标阈值和有限体积仿真共同造成非光滑、非凸、无解析梯度的黑盒目标。

推荐路线是：

```text
外层方向配置 s 枚举/筛选
  + 内层约束混合变量 BO 优化 q
  + NOMAD/MADS 或 DFO 局部抛光
  + 高保真复验
```

其中，混合变量 BO 用于提高昂贵仿真的样本效率，NOMAD/MADS/DFO 用于处理非光滑局部改进，高保真复验用于确保最终排序不依赖低保真或代理模型预测。

### 4.2 `optimization v2.md`

根据调研结论，本周制定了：

```text
methodology/optimization v2.md
```

该文档将优化目标写成：

```text
J = lambda1*J1 + lambda2*J2 + lambda5*J5 + lambdaB*J_B + lambdaR*J_R
```

并明确当前 V2 优化具有以下性质：

- `s` 是离散变量，`q` 是连续变量；
- 目标函数必须通过完整 Bellman--守恒律仿真得到；
- 目标非光滑，不适合直接依赖梯度；
- 方向-速率一致性、关闭通道容量为 0、双向总容量等规则是硬约束，应在候选生成阶段保证；
- 最终结论必须基于高保真复验，而不是低保真筛选或代理模型排序。

文档还规划了完整实验与消融设计：

- BO + DFO 主实验；
- BO 无 DFO；
- 无低保真筛选；
- 不含 `J_B`；
- only-s；
- only-q；
- random search；
- 权重敏感性后处理；
- 候选库合并和高保真排序。

这部分直接成为后续 G5 实验矩阵的实现依据。

## 7. 当前结论

本周最重要的结论不是“换一个优化算法”，而是识别出原控制变量 `eta` 不适合作为当前主优化变量。G4 对比实验说明 SA-HBO 没有超过 random search 和 pure SA；`20260514.md` 的分析进一步指出，问题根源在于 `eta` 连续块贡献弱、`J2` 硬阈值量级过小、离散方向搜索局部性强。

基于入口流入速率控制文献调研，模型已转向更具可实施性的 `z=(s,q)`：`s` 管方向规则，`q` 管内部入口每单位时间允许通过多少人，`eta0` 只作为固定几何引导结构项。`model v2.md` 和 `optimization v2.md` 已经把这一转向形式化，并给出可实现的混合变量黑盒优化路线。

代码层面，G5 v2 已经从单实验 runner 扩展到完整实验矩阵，且具备 TOML 配置、并行执行、对比基线、消融实验和高保真复验输出。当前仍需等待全量矩阵完成后，再给出最终“优化是否优于基线”的实验结论。
