# 宏观模型运行原理时序图

## 1. 宏观模型数学原理时序图

此图展示Hughes连续模型与各向异性引导相结合的数学原理。

```mermaid
sequenceDiagram
    autonumber

    participant 初始条件 as 初始条件<br/>ρ₀(x), 场景几何
    participant 速度密度关系 as 速度-密度关系<br/>v = f(ρ)
    participant Bellman求解器 as HJB/Bellman求解器<br/>求解最优路径
    participant 方向恢复 as 方向恢复<br/>u(x) = argmin
    participant 连续性方程 as 连续性方程<br/>∂ₜρ + ∇·(ρv) = 0
    participant 阶段转移 as 阶段转移<br/>概率分流
    participant 输出 as 输出与记录

    Note over 初始条件: t = 0<br/>初始密度分布 ρ(x,0) = ρ₀(x)<br/>场景几何: walkable, obstacles, exits

    loop 时间步进循环 (t < T)
        初始条件->>速度密度关系: $$\rho(x,t)$$
        Note right of 速度密度关系: Greenshields模型<br/>v = vₘₐₓ(1 - ρ/ρₘₐₓ)
        速度密度关系->>Bellman求解器: v(x,t)

        alt 当前步求解Bellman
            Note over Bellman求解器: HJB方程<br/>∇φ · u = -1/f(ρ)<br/>u ∈ U(x) 受方向约束
            Bellman求解器->>Bellman求解器: Dijkstra-like标签设定<br/>φ(y) = min[φ(x) + h/(s·√(uᵀMu))]
            Bellman求解器->>方向恢复: 值函数 φ(x,t)

            Note over 方向恢复: 从φ恢复最优控制<br/>u* = argmin[φ(x+hu) + cost]
            方向恢复->>连续性方程: 最优方向 u(x,t)<br/>合成速度场 v⃗ = v·u
        else 复用上一步方向 (step % bellman_every ≠ 0)
            速度密度关系->>连续性方程: 复用 u(x,t-Δt)
        end

        连续性方程->>连续性方程: Godunov迎风格式<br/>ρⁿ⁺¹ = ρⁿ - Δt·∇·(ρv⃗)
        Note right of 连续性方程: CFL条件限制Δt<br/>Courant数 < 1

        opt 存在多阶段转移
            连续性方程->>阶段转移: ρ by group
            Note over 阶段转移: 决策区概率分流<br/>Q = κ·χ·ρ·p
            阶段转移->>阶段转移: 更新各群体密度
            阶段转移->>输出: 各阶段人口统计
        end

        连续性方程->>初始条件: ρ(x, t+Δt)
        初始条件->>输出: 记录时序数据<br/>密度场、流量、 evac时间
    end

    Note over 输出: 仿真结束<br/>生成summary.json<br/>生成timeseries.csv
```

## 3. 核心算法详细时序图

### 3.1 Bellman求解器内部流程

```mermaid
sequenceDiagram
    autonumber

    participant Caller as 调用者
    participant Bellman as solve_bellman()
    participant PythonImp as _solve_bellman_python<br/>/ _solve_bellman_optimized
    participant Heap as 优先队列<br/>heapq

    Caller->>Bellman: solve_bellman(walkable, exit_mask, allowed_mask,<br/>speed, step_factor, f_eps, backend)
    activate Bellman

    alt backend == "python"
        Bellman->>PythonImp: _solve_bellman_python(...)
    else backend == "optimized"
        Bellman->>PythonImp: _solve_bellman_optimized(...)
    end
    activate PythonImp

    PythonImp->>PythonImp: 初始化phi = inf<br/>exit_mask位置phi = 0

    loop for (y,x) in exit_mask
        PythonImp->>Heap: heappush(queue, (0.0, y, x))
    end

    loop while queue not empty
        PythonImp->>Heap: heappop(queue) → (value, y, x)

        opt value > phi[y,x]
            Note over PythonImp: 跳过已更新节点
        end

        loop for k in 0..7 (8 directions)
            PythonImp->>PythonImp: 计算邻居(py, px) = (y-dy[k], x-dx[k])

            opt 邻居越界或不可行
                Note over PythonImp: 跳过
            end

            opt (allowed_mask[py,px] & bit[k]) == 0
                Note over PythonImp: 方向不允许，跳过
            end

            PythonImp->>PythonImp: candidate = value +<br/>step_factor[py,px,k]/speed[py,px]

            opt candidate < phi[py,px] - 1e-12
                PythonImp->>PythonImp: phi[py,px] = candidate
                PythonImp->>Heap: heappush(queue, (candidate, py, px))
            end
        end
    end

    PythonImp->>PythonImp: phi[finite] -= min(phi[finite])<br/>归零最小值

    PythonImp-->>Bellman: phi
    deactivate PythonImp

    Bellman-->>Caller: phi
    deactivate Bellman
```

### 3.2 密度更新详细流程

```mermaid
sequenceDiagram
    autonumber

    participant Runner as runner.py
    participant Update as update_density()
    participant Flux as compute_face_fluxes()

    Runner->>Update: update_density(rho, walkable, exit_mask,<br/>vx, vy, dx, dt)
    activate Update

    Update->>Flux: compute_face_fluxes(rho, vx, vy)
    activate Flux

    Note over Flux: vx_face[i,j] = 0.5*(vx[i,j] + vx[i,j+1])<br/>vy_face[i,j] = 0.5*(vy[i,j] + vy[i+1,j])

    Flux->>Flux: 迎风格式选择<br/>rho_x = rho[上游] if vx>=0 else rho[下游]

    Flux->>Flux: fx = vx_face * rho_x<br/>fy = vy_face * rho_y

    Flux-->>Update: (fx, fy)
    deactivate Flux

    Update->>Update: div_x[:,1:-1] = (fx[:,1:] - fx[:,:-1])/dx<br/>div_y[1:-1,:] = (fy[1:,:] - fy[:-1,:])/dx

    Update->>Update: updated = rho - dt*(div_x + div_y)

    Update->>Update: updated[~walkable] = 0<br/>updated[exit_mask] = 0 (sink)

    Update->>Update: sink_mass = sum(updated[exit_mask])*dx*dx<br/>updated[exit_mask] = 0

    Update-->>Runner: (updated, fx, fy, sink_mass)
    deactivate Update
```

## 4. 配置编译详细时序图

### 4.1 场景编译流程

```mermaid
sequenceDiagram
    autonumber

    participant Workflow as config_workflow.py
    participant Compiler as config_compiler.py
    participant Spec as SceneSpec

    Workflow->>Compiler: compile_scene(scene_spec, cfg)
    activate Compiler

    Compiler->>Compiler: walkable = ones((ny, nx))<br/>block_boundaries处理边界

    loop for region in scene_spec.regions
        Compiler->>Compiler: region_masks[name] =<br/>_rect_mask(cfg, x0,x1,y0,y1)
    end

    loop for obstacle in scene_spec.obstacles
        Compiler->>Compiler: walkable[region_masks[obstacle]] = False
    end

    loop for exit in scene_spec.exits
        Compiler->>Compiler: exit_masks[name] =<br/>_selector_mask(exit.regions)
        Compiler->>Compiler: exit_union |= exit_masks[name]
    end

    loop for channel in scene_spec.channels
        Compiler->>Compiler: channel_masks[name] =<br/>_selector_mask(channel.regions)
        Compiler->>Compiler: probe_x[name] =<br/>_channel_probe_x(mask)
    end

    Compiler->>Compiler: 计算centers_y<br/>从channel_masks

    Compiler->>Compiler: 构建BaseScene对象<br/>包含walkable/exit/channel等

    Compiler-->>Workflow: CompiledSceneBundle对象<br/>(scene, region_masks, exit_masks)
    deactivate Compiler
```

### 4.2 案例编译流程

```mermaid
sequenceDiagram
```

## 5. 图例说明

### 时序图符号约定

| 符号 | 含义 |
|------|------|
| `->>` | 同步调用/请求 |
| `-->>` | 返回结果 |
| `activate`/`deactivate` | 对象生命周期 |
| `loop` | 循环执行 |
| `opt` | 可选执行（条件分支） |
| `alt`/`else` | 互斥条件分支 |
| `Note over` | 注释说明 |
| `participant` | 参与对象 |
