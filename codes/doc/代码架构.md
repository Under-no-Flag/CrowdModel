# 代码架构设计

## 1. 架构总览

当前代码已实现**配置驱动的仿真架构**，支持从TOML配置文件加载场景、人口和路线配置，并运行宏观人群动力学仿真。

### 1.1 已实现功能

1. **纯配置场景**：通过`scene.toml`定义几何区域、障碍物、出口和通道
2. **多群体初始分布**：通过`population.toml`配置多群体、多区域的初始密度
3. **多阶段路线配置**：通过`routes.toml`定义阶段、控制策略和阶段转移
4. **配置驱动入口**：`simulate_from_config.py`统一入口支持完整配置工作流

### 1.2 当前限制

1. **图片场景**：尚未实现PNG图片自动栅格化
2. **持续入流**：仅支持初始密度，暂不支持时间分段入流边界
3. **复杂障碍**：当前仅支持矩形区域，多边形/折线障碍需扩展

## 4. 核心数据流

### 4.1 配置→运行时对象转换

```
TOML配置文件
     │
     ▼
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  scene.toml │───▶│ SceneSpec   │───▶│ BaseScene   │
└─────────────┘    └─────────────┘    └─────────────┘
                                            │
┌─────────────┐    ┌─────────────┐         │
│population.  │───▶│ Population  │─────────┤
│    toml     │    │    Spec     │         │
└─────────────┘    └─────────────┘         │    ┌─────────────┐
                                           ├───▶│   CaseModel │
┌─────────────┐    ┌─────────────┐         │    └─────────────┘
│  routes.toml│───▶│  RouteSpec  │─────────┘
└─────────────┘    └─────────────┘
```

### 4.2 运行时数据结构

#### BaseScene（场景几何）
- `walkable`: 可行域掩码
- `initial_rho`: 初始密度场
- `exit_mask`: 出口掩码
- `channel_masks`: 通道观测区掩码字典

#### GroupModel（单群体模型）
- `key`: (stage_id, route_variant) 群体标识
- `goal_mask`: 目标区域掩码
- `sink_mask`: 汇点掩码（退出区域）
- `allowed_mask`: 允许方向掩码（离散8方向）
- `m11/m12/m22`: 各向异性度量张量M(x)

#### CaseModel（完整案例）
- `case_id/title`: 案例标识
- `groups`: 群体字典 {GroupKey: GroupModel}
- `transitions`: 阶段转移规则元组
- `initial_group_density`: 初始群体密度分布

## 5. 数值算法核心

### 5.1 core.py 关键函数

| 函数 | 功能 |
|------|------|
| `solve_bellman()` | 离散Bellman方程求解（Dijkstra-like标签设定） |
| `recover_optimal_direction()` | 从值函数恢复最优控制方向 |
| `update_density()` | 连续性方程显式更新（一阶迎风格式） |
| `compute_cfl_dt()` | CFL条件自适应时间步长 |
| `precompute_step_factors()` | 预计算度量依赖的步长因子 |
| `apply_fixed_probability_splitting()` | 固定概率阶段转移 |

### 5.2 求解器特性

- **8方向离散控制**：E/W/N/S/NE/NW/SE/SW
- **各向异性度量张量**：M(x) = α·τ⊗τ + β·n⊗n
- **方向约束**：通过`allowed_mask`限制局部可行方向
- **多群体支持**：异质人群通过GroupKey区分
- **阶段转移**：决策区内的固定概率分流

## 6. 仿真循环流程

```
simulate_case() 主循环:

初始化:
  ├── 构建群体字典 groups
  ├── 初始化群体密度 rho_by_group
  ├── 预计算步长因子 step_factors
  └── 初始化统计记录 stats

循环 (step < steps && time < time_horizon):
  1. 计算总密度 rho_tot = sum(rho_by_group)
  2. 计算速度 speed = f(rho_tot) [Greenshields]

  3. [若 step % bellman_every == 0]
     对每个群体:
       ├── 求解Bellman: phi = solve_bellman(...)
       └── 恢复方向: (ux, uy) = recover_optimal_direction(...)

  4. 计算速度场: vx = speed * ux, vy = speed * uy

  5. 计算CFL时间步长 dt

  6. 对每个群体更新密度:
     └── rho_next = update_density(rho, vx, vy, dt)

  7. [若存在transitions]
     └── 应用阶段转移: apply_fixed_probability_splitting(...)

  8. 记录统计信息
  9. [若 save_every] 保存快照PNG
  10. time += dt, step += 1

输出:
  ├── timeseries.csv    # 时间序列数据
  ├── timeseries.png    # 时间序列图表
  ├── snapshot_*.png    # 密度场快照
  └── summary.json      # 汇总指标
```

## 7. 配置文件体系

### 7.1 配置层级关系

```
run.toml           # 入口配置，引用其他文件
 ├── simulation    # 数值参数(nx, ny, dx, steps, vmax...)
 ├── scene         # 场景文件路径
 ├── population    # 人口文件路径
 ├── routes        # 路线文件路径
 ├── objective     # 目标函数权重
 └── output        # 输出根目录

scene.toml         # 场景几何配置
 ├── regions       # 命名区域定义（矩形）
 ├── obstacles     # 障碍物区域名称列表
 ├── exits         # 出口定义
 └── channels      # 通道观测区定义

population.toml    # 人口初始分布
 └── initial_groups # 初始群体列表（区域+密度+阶段）

routes.toml        # 路线与控制策略
 ├── case_id/title # 案例标识
 ├── stages        # 阶段定义（目标、控制、转移）
 └── [transitions] # 阶段间转移规则
```

### 7.2 控制模式（routes.toml）

控制按顺序应用，后应用的控制覆盖先应用的：

| 模式 | 参数 | 说明 |
|------|------|------|
| `identity` | - | 无修改 |
| `isotropic` | value | 重置为各向同性张量 |
| `fixed_direction` | direction, alpha, beta | 固定首选方向 |
| `target_region` | target_region, alpha, beta | 指向区域中心 |
| `target_point` | target_point=[x,y], alpha, beta | 指向固定点 |

## 8. 与旧版架构的演进

### 8.1 已完成的迁移

| 旧架构 | 新架构 |
|--------|--------|
| scenes.py中的硬编码场景函数 | scene.toml配置 + compile_scene() |

| case函数内写死的GroupModel | routes.toml配置 + compile_case() |

### 8.2 保留的兼容层

- `CaseModel`保留单群体字段（m11/m12/m22/allowed_mask）用于向后兼容
- `_build_group_models()`自动处理legacy单群体案例
- `_build_initial_group_density()`支持legacy初始密度加载
