## 2. 当前实验目标

当前阶段的实验目标不是直接完成完整优化研究，而是先完成一个可复现、可解释、可扩展的实验闭环，具体包括：

1. 验证模型中的核心机制是否真的改变了人群的局部决策与流动形态，并且这种改变与真实场景中的行为特征一致：
   - 几何引导 `M(x)` 是否改变局部方向偏好、接近角和通道吸引域
   - 方向约束 `U(x)` 是否改变局部最优方向、转向时机和通道选择结构，并在双向来流下抑制逆行与迎面对冲
   - 多阶段固定概率分流是否改变阶段切换、局部汇聚与离场路径组织
2. 比较不同通道管控策略在效率、安全与均衡性上的差异。
3. 验证多阶段行为链是否会改变阶段密度演化。
4. 完成第一版最优控制参数求解设计，使控制变量能够进入自动搜索而不是只靠手工挑选。
5. 为下一阶段的参数敏感性、鲁棒性和完整优化器实现提供标准化实验接口。

## 4. 场景与实验对象

## 4.1 场景 A：三通道主场景

该场景用于机制验证和主策略对比。其几何和实现基础来自当前 `codes/crowd_bellman/scenes.py` 中的三通道场景生成逻辑。

### 场景特征

- 左侧为初始人群区域
- 右侧为出口区域
- 中部隔墙上设置上、中、下三条通道
- 可在通道及其引导区域上施加：
  - 几何引导张量 `M(x)`
  - 单向方向约束 `U(x)`

### 当前已有 case

- `case1_baseline`
- `case2_middle_guided`
- `case3_top_guided`
- `case4_bottom_guided`

### 当前仍需补充的机制 case

- `M-only` case：仅有几何引导，无单向约束
- `U-only` case：仅有单向约束，无几何引导

这两个 case 主要服务于实验组 1，不要求 Day 1 当天实现，但必须在设计中预留。

## 4.2 场景 B：多阶段游览场景

该场景用于多阶段分流行为验证。其实现基础来自当前 `case5_multistage_tour`。

### 场景特征

- 左侧区域为起始集聚区
- 人群先进入右侧平台区域
- 在平台内按阶段变化目标
- 在离场阶段按固定概率分流到不同出口通道
- 最终返回左侧出口

### 当前已有 case

- `case5_multistage_tour`

### 当前需要补充的对照 case

- 单阶段近似 case：忽略阶段转换与分流，将目标近似为单一出口
- 无分流 case：保留阶段但仅设置单一路径

这些对照 case 不再用于论证“分流概率值得被优化”，而是用于识别：

- 多阶段行为层本身是否有必要保留；
- 偏好参数 `p` 是否应作为外生行为结构被识别和固定；
- 若忽略这些行为结构，后续对可控策略的评估是否会失真。

## 7. 各实验组正式设计

## 7.1 G1：模型机制验证

### 目标

G1 不再以“某个机制是否提升系统指标”为第一目标，而是回答两个更基础的问题：

1. 机制真实性：
   - `U(x)` 和 `M(x)` 是否真的改变了人群在局部的“怎么走、何时转向、朝哪个通道靠拢”；
   - 这种改变是否与外滩通道场景中可观察到的行为特征一致。
2. 管理可用性：
   - 当通道方向配置或引导配置改变时，模型是否会产生足够敏感、足够可解释的行为响应；
   - 如果机制错误或过于生硬，后续优化是否会落在错误的人群行为模型上。

因此，G1 的任务是先证明模型“行为上可信、配置上敏感”，再谈它能否作为优化器的评估器。

### 研究问题

1. 单向规则 `U(x)` 是否真的重塑了通道口附近的局部最优方向、转向方式和通道选择结构？如果引入双向来流，它是否还能抑制中通道逆行与迎面对冲？
2. 各向异性 `M(x)` 是否改变了通道口周围的接近角、提前对齐距离、流线弯折方式和通道吸引域？
3. `U+M` 联合作用下，模型是否能对不同通道配置给出既符合规则、又具有真实局部流型的行为响应？
4. 如果去掉 `U` 或 `M` 其中之一，后续策略比较与优化评估是否会建立在失真的行为机制之上？

### 对比 case

| 机制主线 | 方案 | 说明 | 当前状态 |
|------|------|------|------|
| A0 | `case1_baseline` | 单向来流基线，无 `U`，无 `M` | 已有 |
| A1 | `U-only` case | 单向来流下仅启用单向约束，用于看规则是否改变局部方向组织与通道选择 | 已有 |
| A2 | `case_u_bidirectional_baseline` / `case_u_bidirectional_middle_rule` | 双向来流下的 `U` 对冲抑制对照，用于直接检验是否压制中通道逆行与迎面对冲 | 本次补充 |
| B0 | `case1_baseline` | 基线，无各向异性引导 | 已有 |
| B1 | `M-only` case | 仅启用几何引导，用于看接近与转向形态是否变化 | 已有 |
| C0 | `case1_baseline` | 无 `U`、无 `M` 的配置敏感性基线 | 已有 |
| C1 | `U-only` case | 只有规则，无几何引导 | 已有 |
| C2 | `case2_middle_guided` 或新建 `M+U-middle` | `U+M` 联合模型，中通道双控 | 已有基础，可直接使用或细分 |
| C3 | `case5_multistage_tour` | 多阶段行为链展示，用于说明机制验证可自然衔接到行为层 | 已有 |

### 自变量

- 是否启用单向约束 `U`
- 是否启用几何引导 `M`
- 通道方向配置方案
- 各向异性强度或引导参数
- 是否启用多阶段分流机制（作为行为层补充验证）

### 因变量

- 局部最优方向场
- 方向一致性指数 / 逆规则方向占比
- 双向来流下的中通道逆行占比 / 对冲暴露
- 通道吸引域或捕获域
- 接近角分布
- 提前对齐距离
- 流线弯折形态 / 出入口扇形流型
- 关键通道流量占比与局部密度分布

这里的主因变量优先是“行为形态指标”，而不是 `J1/J2/J5`。

### 控制变量

- 场景几何
- 初始密度
- 网格与时间步规则
- 速度函数
- 出口边界与障碍设置
- 观察区域与局部放大窗口
- 统计口径（入口截面、通道轴线、方向场采样方式）
- 单向来流验证与双向对冲验证应分开统计，不混用同一口径

### 主要输出

- 局部方向场图
- 通道吸引域 / 捕获域分区图
- 入口接近角分布图
- 双向来流下的中通道逆行占比 / 对冲暴露对比图
- 局部流线与典型时刻密度放大图
- 机制对比表
- 配置变化下的行为响应对比图

### 判定方式

G1 的判定分成两层：

#### 第一层：机制真实性

- 若 `U-only` 明显改变通道内方向一致性、逆规则方向占比和局部方向图，则说明 `U(x)` 确实改变了局部 Bellman 最优控制；
- 若在双向来流下，加入中通道单向规则后，中通道内的逆行占比、迎面对冲暴露和相向重叠显著下降，则才能进一步说明 `U(x)` 不只是“改路”，而是真的抑制了逆行和对冲；
- 若 `M-only` 明显改变接近角分布、提前对齐位置、流线收敛方式和通道吸引域形状，则说明 `M(x)` 改变的是合法空间内的方向偏好，而不是简单的几何封堵；
- 若这些变化与真实场景经验一致，例如“更早对齐通道轴线”“入口前平滑汇聚而不是临门急转”，则说明机制具有行为合理性。

#### 第二层：管理可用性

- 若切换通道配置后，`U+M` 模型能给出明显、可解释的局部行为重组，则说明该模型足以作为后续策略评估器；
- 若去掉 `U` 或 `M` 后，模型对配置变化的响应变得迟钝、生硬或与场景直觉不符，则说明后续优化不能建立在残缺机制上；
- 因此，G1 的结论不是“先证明 `J1/J2/J5` 更优”，而是先证明“如果机制不对，后续优化就在错误的人群行为模型上做优化”。

## 7.2 G2：管控策略响应与可优化性验证

### 目标

G2 的目标不再是简单回答“哪组策略最好”，而是证明在完整行为模型下：

1. 不同通道配置 `s` 确实会引起可区分、可解释的系统响应；
2. 效率 `J1`、安全 `J2`、均衡 `J5` 之间存在非一致最优，因此后续优化是必要的；
3. 这些响应差异足够稳定，能够形成后续优化器的候选解空间和评估基线。

### 研究问题

1. 不同通道配置是否会系统性改变各通道累计流量、峰值负载时刻和高密度区的位置迁移？
2. `J1/J2/J5` 之间是否存在明确 trade-off，而不是由某一个策略全面支配？
3. 控制变量 `s` 是否已经形成可排序、可优化的响应面，从而说明后续优化器不是装饰而是必要组件？

### 对比 case

| 策略组 | case | 说明 |
|------|------|------|
| S0 | `case1_baseline` | 无管控基线，仅作为响应面对照点 |
| S1 | `middle × eta` 扫描族 | 中通道引导 + 单向约束，并对 `eta` 做候选扫描 |
| S2 | `top × eta` 扫描族 | 上通道引导 + 单向约束，并对 `eta` 做候选扫描 |
| S3 | `bottom × eta` 扫描族 | 下通道引导 + 单向约束，并对 `eta` 做候选扫描 |

其中第一版建议使用：

- `eta ∈ {1, 2, 4, 8, 16, 32}`
- `beta` 固定，仅通过调整 `alpha = eta * beta` 扫描各向异性强度

### 自变量

- 通道配置 `s`
- 受控通道位置
- 几何引导方向
- 单向约束布置
- 各向异性强度 `eta`

### 因变量

- `J1` 对应效率指标
- `J2` 对应安全指标
- `J5` 对应均衡性指标
- 各通道累计流量 `F_c`
- 各通道峰值负载时刻
- 高密度热点位置 / 迁移轨迹
- 累计通过量

### 控制变量

- 相同场景几何
- 相同初始人群
- 相同速度函数与数值参数
- 相同仿真时长

### 主要输出

- 主实验汇总表
- `J1/J2/J5` Pareto 散点图
- `s × eta` 的 `J1/J2/J5` 响应面 / 热力图
- 通道累计流量与占比图
- 峰值负载时刻对比图
- 高密度热点迁移图
- 候选策略排序表

### 判定方式

G2 不要求某一策略在所有目标上都最优，而是重点判断：

- 不同配置是否产生结构化且可解释的响应差异；
- 是否存在“一组策略提升效率但牺牲安全”或“降低风险但负载更不均衡”的现象；
- 若没有单一全面支配策略，则说明后续优化是必要的，而不是靠经验即可稳定选出最优方案。

## 7.3 G3：多阶段行为层必要性与偏好参数识别支持

### 目标

G3 的目标不再是论证“固定概率分流本身值得被优化”，而是证明：

1. 多阶段行为层必须被保留，否则后续策略评估会失真；
2. 偏好参数 `p` 应作为外生行为结构被识别和固定，而不是与可控策略混为一体；
3. 在固定 `p` 的前提下，模型仍然能够支撑后续对可控策略 `s`（必要时含 `eta`）的优化。

### 研究问题

1. 与单阶段近似相比，多阶段行为链是否会改变平台滞留、出口负载和热点迁移？
2. 若保留多阶段结构但去掉出口偏好差异，模型输出是否会与完整行为层显著不同？
3. 若忽略多阶段行为层或固定偏好参数 `p`，后续对策略优劣的判断是否会出现偏差？

### 对比 case

| 行为层级 | case | 说明 | 当前状态 |
|------|------|------|------|
| A0 | 单阶段近似 case | 忽略“上平台-游览-离场-回流”的阶段转换，只保留简化起终点 | 本次补充 |

## 7.4 G4：最优控制参数求解

### 目标

对可控参数进行第一版自动求解，回答“哪些管控参数组合最优”，并为后续升级到 SA-HBO 提供实验基线与软件接口。

### 研究问题

1. 在给定目标函数下，自动搜索是否能找到优于人工基线的管控方案？
2. 离散策略选择、几何引导强度和分流概率三类变量中，哪些对结果最敏感？
3. 当 `λ1/λ2/λ5` 权重改变时，最优参数组合是否发生迁移？

### 优化变量

第一阶段只优化下列可落地变量：

- 离散变量 `s`
  - 受控策略编号
  - 候选集合可取 `{baseline, middle, top, bottom}`
- 连续变量 `eta`
  - 各向异性比值 `eta = alpha / beta`
  - 第一阶段按有限候选值搜索
- 连续变量 `p`
  - 固定分流概率
  - 第一阶段按候选概率集合搜索

第一阶段暂不纳入：

- 决策区域位置 `xi`
- 完整方向配置组合搜索
- 完整 SA-HBO

### 优化目标

采用加权和目标：

$$
J = \lambda_1 J_1 + \lambda_2 J_2 + \lambda_5 J_5,
$$

其中：

- `J1`：总旅行时间近似量
- `J2`：高密度暴露时间
- `J5`：通道负载方差

### 第一版求解方法

第一版不直接实现 `methodology/optimization.md` 中的完整 SA-HBO，而采用更适合当前进度的结构化候选搜索：

1. 枚举离散策略变量 `s`
2. 对 `eta` 做粗网格搜索
3. 对 `p` 做有限候选集合搜索
4. 对每组参数运行仿真并计算目标函数 `J`
5. 输出最优参数组合与候选排序表

### 对比 case

| 优化层级 | 对比对象 | 说明 |
|------|------|------|
| O0 | 人工基线策略 | 例如 `case1_baseline` 与人工指定策略 |
| O1 | 离散策略搜索 | 在已有主场景策略集合中选最优 |
| O2 | `eta` 粗搜索 | 在固定策略下搜索几何引导强度 |
| O3 | `p` 搜索 | 在多阶段场景下搜索固定概率分流 |
| O4 | 联合搜索 | 对 `s + eta + p` 的简化组合搜索 |

### 因变量

- 综合目标函数 `J`
- `J1/J2/J5`
- 峰值密度
- 累计通过量
- 通道负载占比

### 控制变量

- 场景几何
- 网格与时间步规则
- 速度函数
- 初始密度
- 未被优化的参数默认值

### 主要输出

- 最优参数配置表
- 候选策略排序表
- 不同权重下的最优解对比表
- 参数搜索日志

### 判定方式

- 若搜索得到的最佳方案在 `J` 上优于人工基线，则说明参数求解链路有效；
- 若不同权重下的最优参数发生迁移，则说明该问题确实存在多目标权衡；
- 若 `eta` 与 `p` 的改变会显著影响 `J1/J2/J5`，则说明这些参数值得进入后续正式优化框架。
