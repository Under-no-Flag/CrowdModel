## 2. 当前实验目标

当前阶段的实验目标不是直接完成完整优化研究，而是先完成一个可复现、可解释、可扩展的实验闭环，具体包括：

1. 验证模型中的三个核心机制是否分别产生可识别的作用：
   - 几何引导 `M(x)`
   - 方向约束 `U(x)`
   - 多阶段固定概率分流
2. 比较不同通道管控策略在效率、安全与均衡性上的差异。
3. 验证多阶段行为链是否会改变阶段密度演化和出口负载分配。
4. 完成第一版最优控制参数求解设计，使控制变量能够进入自动搜索而不是只靠手工挑选。
5. 为下一阶段的参数敏感性、鲁棒性和完整优化器实现提供标准化实验接口。

## 4. 场景与实验对象

## 4.1 场景 A：三通道主场景

该场景用于机制验证和主策略对比。其几何和实现基础来自当前 `codes/crowd_bellman/scenes.py` 中的三通道场景生成逻辑。

### 场景特征

- 左侧为初始人群区域
- 右侧为出口区域
- 中部隔墙上设置上、中、下三条通道
- 可在通道及其引导区域上施加：
  - 几何引导张量 `M(x)`
  - 单向方向约束 `U(x)`

### 当前已有 case

- `case1_baseline`
- `case2_middle_guided`
- `case3_top_guided`
- `case4_bottom_guided`

### 当前仍需补充的机制 case

- `M-only` case：仅有几何引导，无单向约束
- `U-only` case：仅有单向约束，无几何引导

这两个 case 主要服务于实验组 1，不要求 Day 1 当天实现，但必须在设计中预留。

## 4.2 场景 B：多阶段游览场景

该场景用于多阶段分流行为验证。其实现基础来自当前 `case5_multistage_tour`。

### 场景特征

- 左侧区域为起始集聚区
- 人群先进入右侧平台区域
- 在平台内按阶段变化目标
- 在离场阶段按固定概率分流到不同出口通道
- 最终返回左侧出口

### 当前已有 case

- `case5_multistage_tour`

### 当前需要补充的对照 case

- 单阶段近似 case：忽略阶段转换与分流，将目标近似为单一出口
- 无分流 case：保留阶段但仅设置单一路径

这些对照 case 用于识别“多阶段”和“分流”本身的效应。

## 7. 各实验组正式设计

## 7.1 G1：模型机制验证

### 目标

验证模型中的三个核心机制是否各自具有必要性，并识别其独立作用与耦合效应。

### 研究问题

1. 仅引入 `M(x)` 是否会显著改变通道利用分布？
2. 仅引入 `U(x)` 是否会减少逆向冲突但带来局部堆积转移？
3. 同时引入 `M(x)+U(x)` 是否会产生额外协同效应？
4. 多阶段分流是否会进一步改变出口负载分配？

### 对比 case

| 机制层级 | 方案 | 说明 | 当前状态 |
|------|------|------|------|
| C0 | `case1_baseline` | 基线，无几何引导，无单向约束 | 已有 |
| C1 | `M-only` case | 仅启用几何引导 | 待补 |
| C2 | `U-only` case | 仅启用单向约束 | 待补 |
| C3 | `case2_middle_guided` 或新建 `M+U-middle` | 中通道双控 | 已有基础，可直接使用或细分 |
| C4 | `case5_multistage_tour` | 多阶段分流机制展示 | 已有 |

### 自变量

- 是否启用几何引导 `M`
- 是否启用单向约束 `U`
- 是否启用多阶段分流

### 因变量

- 平均密度
- 峰值密度
- 通道累计流量
- 通道流量占比
- 速度场间断度

### 控制变量

- 场景几何
- 初始密度
- 网格与时间步规则
- 速度函数
- 出口边界与障碍设置

### 主要输出

- 机制对比表
- 典型时刻密度图
- 通道负载占比图
- 稳定性指标对比图

### 判定方式

- 若 `M-only` 显著改变目标通道流量占比，则说明几何引导机制有效；
- 若 `U-only` 显著改变方向稳定性或局部密度分布，则说明方向约束机制有效；
- 若 `M+U` 与单独启用存在明显差异，则说明机制耦合不是冗余设计；
- 若多阶段分流显著改变出口负载，则说明行为层建模不是装饰项。

## 7.2 G2：主场景策略对比

### 目标

比较不同通道管控策略在效率、安全与均衡性上的综合表现，为后续优化研究提供基准策略集合。

### 研究问题

1. 中通道受控、上通道受控、下通道受控三类策略的效果有何差异？
2. 某些策略是否提升效率但牺牲局部安全？
3. 是否存在“通行效率高但负载极不均衡”的策略？

### 对比 case

| 策略组 | case | 说明 |
|------|------|------|
| S0 | `case1_baseline` | 无管控基线 |
| S1 | `case2_middle_guided` | 中通道引导 + 单向约束 |
| S2 | `case3_top_guided` | 上通道引导 + 单向约束 |
| S3 | `case4_bottom_guided` | 下通道引导 + 单向约束 |

### 自变量

- 受控通道位置
- 几何引导方向
- 单向约束布置

### 因变量

- `J1` 对应效率指标
- `J2` 对应安全指标
- `J5` 对应均衡性指标
- 峰值密度
- 累计通过量

### 控制变量

- 相同场景几何
- 相同初始人群
- 相同速度函数与数值参数
- 相同仿真时长

### 主要输出

- 主实验汇总表
- 三类目标对比图
- 通道流量占比图
- 时间序列对比图

### 判定方式

不要求一组策略在所有目标上都最优，而是识别：

- 哪个策略效率最好
- 哪个策略高密度暴露最低
- 哪个策略通道负载更均衡

这组实验是后续优化器设计的基准参考。

## 7.3 G3：多阶段分流行为验证

### 目标

验证多阶段行为链和固定概率分流机制对总密度演化、阶段转换和出口负载分配的影响。

### 研究问题

1. 与单阶段近似相比，多阶段建模是否改变总密度时空分布？
2. 固定概率分流是否能稳定地产生预期的出口分担比例？
3. 不同阶段目标是否会引起中间区域的局部拥堵迁移？

### 对比 case

| 行为层级 | case | 说明 | 当前状态 |
|------|------|------|------|
| B0 | 单阶段近似 case | 忽略阶段切换 | 待补 |
| B1 | 无分流 case | 保留阶段但仅单一路径 | 待补 |
| B2 | `case5_multistage_tour` | 多阶段 + 固定概率分流 | 已有 |

### 自变量

- 是否分阶段
- 是否分流
- 分流概率
- 切换率 `kappa`

### 因变量

- 各阶段群体质量变化
- 各出口累计流量
- 总旅行时间近似量
- 高密度暴露时间
- 峰值密度

### 控制变量

- 平台几何
- 起始密度
- 共享拥堵函数
- 相同数值格式与参数

### 主要输出

- 分阶段密度演化图
- 出口分流占比图
- 阶段质量守恒检查
- 多阶段行为对比表

### 判定方式

- 若多阶段 case 的出口负载分配与单阶段近似有明显差异，则说明行为链建模具有必要性；
- 若固定概率分流输出与设定概率整体一致，说明阶段转移机制实现正确；
- 若阶段切换引起局部密度迁移，说明行为层会反馈到宏观拥堵结构。

## 7.4 G4：最优控制参数求解

### 目标

对可控参数进行第一版自动求解，回答“哪些管控参数组合最优”，并为后续升级到 SA-HBO 提供实验基线与软件接口。

### 研究问题

1. 在给定目标函数下，自动搜索是否能找到优于人工基线的管控方案？
2. 离散策略选择、几何引导强度和分流概率三类变量中，哪些对结果最敏感？
3. 当 `λ1/λ2/λ5` 权重改变时，最优参数组合是否发生迁移？

### 优化变量

第一阶段只优化下列可落地变量：

- 离散变量 `s`
  - 受控策略编号
  - 候选集合可取 `{baseline, middle, top, bottom}`
- 连续变量 `eta`
  - 各向异性比值 `eta = alpha / beta`
  - 第一阶段按有限候选值搜索
- 连续变量 `p`
  - 固定分流概率
  - 第一阶段按候选概率集合搜索

第一阶段暂不纳入：

- 决策区域位置 `xi`
- 完整方向配置组合搜索
- 完整 SA-HBO

### 优化目标

采用加权和目标：

$$
J = \lambda_1 J_1 + \lambda_2 J_2 + \lambda_5 J_5,
$$

其中：

- `J1`：总旅行时间近似量
- `J2`：高密度暴露时间
- `J5`：通道负载方差

### 第一版求解方法

第一版不直接实现 `methodology/optimization.md` 中的完整 SA-HBO，而采用更适合当前进度的结构化候选搜索：

1. 枚举离散策略变量 `s`
2. 对 `eta` 做粗网格搜索
3. 对 `p` 做有限候选集合搜索
4. 对每组参数运行仿真并计算目标函数 `J`
5. 输出最优参数组合与候选排序表

### 对比 case

| 优化层级 | 对比对象 | 说明 |
|------|------|------|
| O0 | 人工基线策略 | 例如 `case1_baseline` 与人工指定策略 |
| O1 | 离散策略搜索 | 在已有主场景策略集合中选最优 |
| O2 | `eta` 粗搜索 | 在固定策略下搜索几何引导强度 |
| O3 | `p` 搜索 | 在多阶段场景下搜索固定概率分流 |
| O4 | 联合搜索 | 对 `s + eta + p` 的简化组合搜索 |

### 因变量

- 综合目标函数 `J`
- `J1/J2/J5`
- 峰值密度
- 累计通过量
- 通道负载占比

### 控制变量

- 场景几何
- 网格与时间步规则
- 速度函数
- 初始密度
- 未被优化的参数默认值

### 主要输出

- 最优参数配置表
- 候选策略排序表
- 不同权重下的最优解对比表
- 参数搜索日志

### 判定方式

- 若搜索得到的最佳方案在 `J` 上优于人工基线，则说明参数求解链路有效；
- 若不同权重下的最优参数发生迁移，则说明该问题确实存在多目标权衡；
- 若 `eta` 与 `p` 的改变会显著影响 `J1/J2/J5`，则说明这些参数值得进入后续正式优化框架。

## 8.1 理论指标与代码字段映射

| 理论指标 | 数学含义 | 论文中的表述 | 当前代码字段 | 当前状态 | 说明 |

第一阶段允许先用原始量或基于基线的相对量，只要在代码和结果表中口径一致即可。
